# `<scope>` vs `<define>`

| 구분           | `<scope>`                                                            | `<define>`                                            |
|--------------|----------------------------------------------------------------------|-------------------------------------------------------|
| 목적           | 한 문서 안에서 **DOM + script 범위를 국소적으로 묶기**                               | 외부/내부 문서에 **컴포넌트/모듈(별도 NS)을 정의**                      |
| NS(provider) | **기존 NS 재사용** (새 NS 안 만듦)                                            | **새로운 NS 생성**                                         |
| 사용 시나리오      | 한 페이지에 같은 구조를 여러 개 두고, 각각을 독립된 `영역`처럼 다루고 싶을 때 <br>(퀴즈 박스, 카드, 섹션 등) | 외부 HTML로 배포하는 컴포넌트/위젯, 재사용 가능한 UI 모듈, 별도 디자인 시스템 블록 등 |
| 스크립트에서 특징    | `$inject.scopeElement` 로 자기 `<scope>` 구역을 바로 찾을 수 있음                 | `$task['ns:taskID']`, 템플릿/스타일을 NS 기준으로 분리해서 사용        |

### 용도별로 구분

한 페이지 안에서만 쓰는 로컬 UI 조각
- `<scope>` 로 영역을 나누고, `<script>` 에서 `$inject.scopeElement` 기준으로 DOM 다루기

외부 파일로 떼어내서 여러 프로젝트/페이지에서 재사용하고 싶은 컴포넌트
- 그 외부 파일 안을 `<define ns="quiz"> ... </define>` 형태로 만들고,  
  `<task preload="quiz.html">` 등으로 불러와서 사용
